Gobierno de datos (*Data Governance*)
========================================================================

.. tags:: transparencia,calidad


¿Cuál es la diferencia entre gobernanza de datos y gestión de datos?
--------------------------------------------------------------------

Los términos gobernanza de datos y gestión de datos suelen usarse
indistintamente, pero tienen propósitos diferentes. La **gobernanza de
datos** establece reglas y políticas sobre el uso de los datos, mientras
que la **gestión de datos** se refiere a cómo se gestionan y utilizan
según las reglas de gobernanza.

Aunque son distintas, la gobernanza y la gestión de datos trabajan
juntas para crear pilares clave de la estrategia de datos a largo plazo
de una organización. Garantizan los estándares de calidad de los datos y
ayudan a crear una base sólida de datos para proporcionar información
fiable.

¿Qué es la gestión de datos?
----------------------------

La gestión de datos abarca los procesos diarios, técnicos y operativos
que garantizan la correcta recopilación, almacenamiento, mantenimiento y
acceso de los datos a los usuarios.

Algunas de las maneras en que las buenas prácticas de gestión de datos
se manifiestan en una organización incluyen las siguientes:

- **Arquitectura de datos**: este marco determina cómo se estructuran y
  gestionan los datos dentro de una organización, incluidos modelos,
  políticas, estándares y reglas.

- **Modelado de datos**: es una demostración de cómo fluyen los datos a
  través de una aplicación u organización para ayudar a los usuarios a
  comprender la estructura de los datos.

- **Ingesta de datos**: Una gestión de datos adecuada también debe
  contemplar la ingesta de datos en los *pipelines*. Esto suele implicar
  el procesamiento de datos para corregir errores, eliminar duplicados y
  combinar conjuntos de datos.

- **Catalogación de datos**: al crear un inventario de sus recursos de
  datos, puede hacer que sean más fáciles de buscar y permitir la
  colaboración entre usuarios.

- **Almacenamiento de datos**: Las organizaciones suelen utilizar un
  almacén de datos, una base de datos, un lago de datos o un *data
  lakehouse*. Su estrategia de gestión de datos determina las soluciones
  de almacenamiento más adecuadas para su empresa u organización,
  teniendo en cuenta la complejidad y el tipo de datos.

- **Optimización y mantenimiento de datos**: El proceso de optimización
  de datos es continuo. A medida que sus datos y su uso cambian con el
  tiempo, es necesario optimizarlos continuamente para mantener un
  rendimiento analítico óptimo.

Las estrategias de gestión de datos son más importantes que nunca. A
medida que aumenta la complejidad y la cantidad de datos, aumenta
también la importancia de tener una estrategia para organizar y analizar
grandes cantidades de datos para las cargas de trabajo de análisis,
inteligencia empresarial e inteligencia artificial en toda la
organización. Los beneficios incluyen:

- **Eficiencia e integridad**. Una buena gestión de datos optimiza los
  procesos y garantiza la integridad de los datos. El procesamiento
  eficiente de los datos garantiza su correcta organización y la
  ausencia de duplicaciones o errores.

- **Fiabilidad y precisión**. Implementar procedimientos exhaustivos de
  gestión de datos y automatizar los flujos de trabajo garantiza
  trabajar con conjuntos de datos óptimos.

- **Escalabilidad y recuperación**. Otra ventaja de la gestión de datos
  es que puede ayudar a su negocio a escalar. Con una mejor visibilidad
  y datos fiables, podrá tomar decisiones más rápido, responder a los
  cambios y aprovechar nuevas oportunidades. También tendrá una mejor
  comprensión de las preferencias de los clientes y podrá demostrar su
  crecimiento y potencial a nuevos inversores.

Una buena gestión de datos también incluye estrategias robustas de
**copia de seguridad y recuperación**. Estas estrategias garantizan la
rápida recuperación de datos críticos y minimizan el tiempo de
inactividad en caso de un ciberataque o un fallo del sistema.

¿Qué es la gobernanza de datos?
------------------------------------------------------------------------

Si la gestión de datos se refiere a las operaciones diarias de gestión
de datos de una organización, la gobernanza de datos se refiere al marco
general de políticas, estándares y procedimientos que definen cómo deben
almacenarse y gestionarse los datos dentro de la organización. La
gobernanza de datos es el proceso de definir la propiedad de los datos,
las funciones y las responsabilidades para la gestión de datos dentro de
la organización, así como la aplicación de las políticas y los
procedimientos de datos.

Una buena gobernanza de datos se alinea con su estrategia empresarial
general y proporciona capacidades de gestión, calidad, visibilidad,
seguridad y cumplimiento normativo en todos los niveles de su
organización. Unas políticas sólidas de gobernanza de datos pueden
determinar cómo aprovechar los activos de datos sin procesar y convertir
esa información en una ventaja competitiva que fomente la fidelización
de los clientes e impulse los ingresos.

Los elementos clave de la gobernanza de datos incluyen los siguientes:

- **Descubrimiento de datos**. El descubrimiento de datos, el primer
paso en cualquier proceso de gobernanza, ayuda a las organizaciones a
comprender los datos recopilados y a transformarlos en información
útil para orientar la estrategia empresarial general. Esto puede
implicar analizar los datos en busca de patrones o tendencias útiles,
o descubrir relaciones entre ellos para obtener información más
precisa.

- **Calidad de los datos**. Una decisión es tan buena como los datos en
los que se basa, y la calidad de los datos es un aspecto clave para
una gobernanza de datos eficaz. Al desarrollar reglas de calidad de
datos, las organizaciones priorizan atributos clave como la
precisión, la integridad, la frescura y el cumplimiento normativo.
Además, la gobernanza de datos garantiza la trazabilidad de los datos
y el seguimiento de posibles cambios.

- **Clasificación de datos**. La clasificación de datos es el proceso
de categorizar los datos según su sensibilidad, valor e importancia.
Una estrategia de gobernanza ayuda a una organización a clasificar e
identificar los datos según su nivel de riesgo y puede contribuir a
mitigar e implementar las medidas y políticas de seguridad
necesarias.

- **Seguridad de datos**. Las políticas de seguridad de datos promueven
  la seguridad de sus datos y pueden proteger a su organización del
  acceso no autorizado. Esto puede incluir la incorporación de
  protocolos de seguridad como el cifrado y la anonimización de datos
  para protegerse contra ciberataques, o conservar únicamente la
  información necesaria de los clientes y mantener registros de su
  consentimiento, garantizando así el cumplimiento de las leyes de
  privacidad de datos y las normativas específicas del sector.

- **Linaje de datos**. Un pilar esencial de la gobernanza de datos es el
  uso del linaje de datos para comprender las fuentes y el consumo de
  datos. Este proceso automatizado captura metadatos y eventos
  relevantes a lo largo del ciclo de vida de los datos, lo que
  proporciona a los usuarios una mejor comprensión de cualquier error en
  los datos y prepara el sistema para auditorías sin el coste de
  desarrollar un registro de auditoría manual.

- **Transparencia de datos**. Una estrategia de gobernanza que promueva
  la transparencia de datos puede generar confianza, rendición de
  cuentas y una mejor toma de decisiones en toda la organización. Los
  procedimientos de gobernanza de datos abiertos y claros promueven una
  mejor colaboración entre equipos y garantizan que todos se adhieran a
  las mismas políticas de gobernanza. También pueden ayudar a
  identificar problemas de cumplimiento normativo, fomentando una
  cultura general de gestión de datos y cumplimiento normativo.

- **Controles de acceso**. Una parte importante de cualquier estrategia
  de gobernanza de datos es garantizar que los datos se utilicen y
  accedan a ellos de forma adecuada y segura. Esto incluye definir las
  funciones y responsabilidades de quienes tienen acceso a los datos,
  así como establecer procesos y procedimientos para otorgar y revocar
  el acceso. Unos controles de acceso sólidos ayudan a proteger los
  datos confidenciales, a la vez que proporcionan a los usuarios
  información más relevante y aplicable a sus funciones y
  responsabilidades.

- **Auditoría de datos**. La auditoría periódica de datos garantiza la
  seguridad, el cumplimiento normativo y la rendición de cuentas, a la
  vez que protege a la organización del uso indebido de datos, como
  filtraciones de datos o incumplimiento normativo. Las auditorías
  también pueden ser útiles para identificar cómo deben cambiar o
  evolucionar las estrategias de gobernanza ante nuevos datos o procesos
  dentro de una organización. Al implementar procedimientos de auditoría
  de datos, las organizaciones también pueden promover y mantener la
  confianza de los clientes, garantizando la protección y el cuidado
  continuos de los datos confidenciales.

Una organización no puede implementar una estrategia de gestión de datos
sin contar con una buena gobernanza de datos. Si bien sus ámbitos de
aplicación son distintos, la gobernanza y la gestión de datos trabajan
en conjunto para proporcionar a la organización un marco y herramientas
para gestionar cada etapa del ciclo de vida de los datos.

Dimensiones de la calidad del dato.
------------------------------------------------------------------------

- **Precisión**: ¿Son los datos correctos y libres de errores?

La precisión se refiere al grado en que los datos reflejan correctamente
la realidad. En otras palabras, ¿los datos son confiables y exactos?
Ejemplo práctico: Una dirección incorrecta en una base de datos puede
provocar envíos fallidos, afectando la experiencia del cliente y
generando costos innecesarios.

- **Completitud**: ¿Están todos los datos esperados presentes?

Este indicador mide el porcentaje de datos que están completos o
disponibles en un conjunto de datos en relación con lo esperado. La
incompletidad de los datos puede afectar directamente la capacidad de
tomar decisiones confiables y precisas.

Ejemplo práctico: En un sistema CRM, campos como “correo electrónico”,
“número de contacto” o “fecha de nacimiento” son críticos. Si el 30 % de
los registros carecen de estos datos, esto podría generar ineficiencias
en las campañas de marketing o en la comunicación con clientes.

- **Consistencia**: ¿Son los datos coherentes en diferentes sistemas y
  ubicaciones?

La consistencia evalúa si los datos son uniformes entre sistemas o
dentro de una base de datos. La falta de consistencia puede generar
discrepancias que afecten los análisis o los informes.

Ejemplo práctico: En un sistema financiero, un cliente puede aparecer
con diferentes saldos en dos bases de datos distintas debido a procesos
de integración defectuosos o duplicidades.

- **Validez**: ¿Los datos cumplen con las reglas y formatos definidos?
Que consiste en que el dato presenta conformidad (formato, tipo,
rango) respecto a su definición.

- **Oportunidad**: ¿Están los datos actualizados y disponibles cuando
se necesitan?

Este indicador mide si los datos están disponibles y actualizados en
el momento en que son necesarios para la toma de decisiones.

Importancia: En sectores como la logística o la atención al cliente,
contar con información desactualizada puede resultar en retrasos,
pérdidas financieras o insatisfacción del cliente.

Ejemplo práctico: Una base de datos de inventarios que no se
actualiza en tiempo real podría provocar ventas de productos que ya
no están disponibles.

Recomendación: Implementar procesos automatizados que mantengan la
frescura de los datos y minimicen el tiempo entre la recopilación y
la actualización.

- **Unicidad**: ¿Hay datos duplicados o redundantes? Que consiste en
que el dato debe guardarse de forma única para evitar
inconsistencias.

- **Consistencia**, que consiste en la ausencia de diferencias al
comparar dos representaciones del mismo dato, evitando información
contradictoria.

Qué entendemos por Calidad de Datos
------------------------------------------------------------------------

- Calidad de Datos: Implementar procesos para garantizar la precisión,
completitud, consistencia, etc. de los datos.

- Seguridad de Datos: Proteger los datos contra accesos no autorizados
y amenazas.

- Almacenamiento y Acceso a Datos: Gestionar el almacenamiento y
proporcionar acceso a los datos de manera eficiente.

- Ciclo de Vida de los Datos: Gestionar los datos desde su creación
hasta su eliminación o archivo.

- Programa de Calidad de Datos: Se entiende por programa de calidad de
datos la metodología estratégica y sistemática para la evaluación de
la calidad de los datos dentro de una organización, la cual permite
identificar atributos de calidad de datos, analizar dichos atributos
en su contexto actual o futuro y proporcionar una guía para mejorar
la calidad de los datos.
